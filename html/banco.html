Banco de Dados

  Em um banco de dados, é essencial entender o conceito e a importância dele. Um banco de dados é uma coleção organizada de informações, ou dados, geralmente armazenados eletronicamente em um sistema de computador.

  Normalmente, o banco de dados é controlado por um sistema de gerenciamento de banco de dados (DBMS) e ele possui dois tipos: O banco de dados Relaciona e o Não-Relacional. No contexto de banco de dados, há dois conceitos importantes: o Modelo Entidade e Relacionamento (MER) e o Diagrama de Entidade e Relacionamento (DER).

  O MER é utilizado para criar um modelo conceitual que representa a estrutura lógica dos dados em um sistema. Ele é composto por entidades e relacionamentos, e pode ser representado textualmente ou visualmente por meio de um diagrama de classe.

  Já o DER é uma representação visual do modelo entidade e relacionamento, que inclui atributos, relacionamentos e cardinalidades de cada elemento. Normalmente, o DER serve como base para projetar e implementar o modelo físico do banco de dados. O modelo físico é a implementação concreta do banco de dados em um Sistema de Gerenciamento de Banco de Dados (SGBD), como o PostgreSQL, por exemplo.

  Após compreendermos esses conceitos, passamos a aprender sobre a linguagem utilizada em um banco de dados. Isso inclui os comandos de DDL (Linguagem de Definição de Dados), DML (Linguagem de Manipulação de Dados), DQL (Linguagem de Consulta de Dados), DCL (Linguagem de Controle de Dados) e DTL (Linguagem de Transação de Dados).

  Para obtermos uma compreensão prática da criação de todo esse processo, desenvolvemos um projeto que envolveu a criação do modelo lógico, conceitual e físico de um banco de dados específico para uma universidade. Nesse projeto, incorporamos todas as entidades e atributos necessários para simular de forma abrangente os elementos essenciais de um banco de dados universitário. Além disso, estabelecemos relacionamentos entre essas entidades, permitindo a troca de informações importantes entre elas.

  Por fim, discutimos a normalização do banco de dados, que é o processo de organização padrão para melhor visualização e compreensão dos dados. Durante esse processo, realizamos a atomicidade dos dados, a criação de chaves primárias e a remoção de atributos repetidos. Além disso, garantimos que não houvesse dependência parcial ou dependências transitivas, ou seja, uma relação não deve ter um atributo não-chave determinado funcionalmente por outro atributo não-chave, onde todos estes processos fazem parte da Primeira, Segunda e Terceira forma normal.


  Nossos últimos projetos de banco de dados tiveram como objetivo aprimorar o domínio das linguagens mencionadas anteriormente, com ênfase especial na DQL. Buscamos aprender a extrair informações específicas de um banco de dados, para isso utilizamos um banco de dados contendo informações sobre os Avengers. Um exemplo das informações que tínhamos que extrair era o personagem com o maior número de aparições nos filmes, o personagem mais jovem e o mais velho, entre outros dados relevantes.

  Além disso, enfrentamos o desafio de normalizar um banco de dados completo que estava fora das formas normais de normalização padrão. Utilizamos o banco de dados do Titanic e aplicamos as três formas normais de normalização, tornando-o organizado e funcional. Essa normalização permitiu que o banco de dados fosse facilmente acessado e consultado, garantindo uma compreensão clara dos dados e um funcionamento adequado.
