Banco de Dados

Em um banco de dados, é essencial entender o conceito e a importância dele. Um banco de dados é uma coleção organizada de informações, ou dados, geralmente armazenados eletronicamente em um sistema de computador. Normalmente, o banco de dados é controlado por um sistema de gerenciamento de banco de dados (DBMS) e ele possui dois tipos: O banco de dados Relaciona e o Não-Relacional.

No contexto de banco de dados, há dois conceitos importantes: o Modelo Entidade e Relacionamento (MER) e o Diagrama de Entidade e Relacionamento (DER). O MER é utilizado para criar um modelo conceitual que representa a estrutura lógica dos dados em um sistema. Ele é composto por entidades e relacionamentos, e pode ser representado textualmente ou visualmente por meio de um diagrama de classe. Já o DER é uma representação visual do modelo entidade e relacionamento, que inclui atributos, relacionamentos e cardinalidades de cada elemento. Normalmente, o DER serve como base para projetar e implementar o modelo físico do banco de dados. O modelo físico é a implementação concreta do banco de dados em um Sistema de Gerenciamento de Banco de Dados (SGBD), como o PostgreSQL, por exemplo.

Após compreendermos esses conceitos, passamos a aprender sobre a linguagem utilizada em um banco de dados. Isso inclui os comandos de DDL (Linguagem de Definição de Dados), DML (Linguagem de Manipulação de Dados), DQL (Linguagem de Consulta de Dados), DCL (Linguagem de Controle de Dados) e DTL (Linguagem de Transação de Dados).

Por fim, discutimos a normalização do banco de dados, que é o processo de organização padrão para melhor visualização e compreensão dos dados. Durante esse processo, realizamos a atomicidade dos dados, a criação de chaves primárias e a remoção de atributos repetidos. Além disso, garantimos que não houvesse dependência parcial ou dependências transitivas, ou seja, uma relação não deve ter um atributo não-chave determinado funcionalmente por outro atributo não-chave.
